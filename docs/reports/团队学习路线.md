## 分工节点（8 月底验收后）

> 前 8 周五人全栈，每人都跑通了移动→物理→UI→射击→AI→网络的完整链路。8 月底验收后正式分方向，每人认领一个主方向加一个副方向。主方向决定日常开发，副方向确保关键节点有人 backup。

### 方向一：前端 · 场景交互（2 人）

**日常职责**

- 场景渲染：多层视差背景（3 层不同速度）、Tilemap 地形边界、序列帧动画（引擎火焰、爆炸）
- 飞行操控：Rigidbody2D 物理参数全调（惯性系数、转向阻尼、速度上下界、加速曲线）
- 武器系统：机炮射线检测、导弹追踪逻辑、碰撞事件触发爆炸特效
- 抬头显示：血量条、速度表、武器冷却图标、雷达小地图
- 界面全流程：主菜单 → 设置 → 房间选择 → 战斗中 → 暂停 → 结算 → 回主菜单
- 素材管理：`assets/` 入库审核，全组素材统一色调、比例、像素密度

**雷达小地图技术要点**

- 先做方形小地图（计算简单），再用 Shader 裁剪成圆形
- 实时映射：敌机红色三角、友军蓝色圆点、导弹黄色小点
- 缩放比例：小地图尺寸与实际场景尺寸的比值可配

**每周额外关注**

- 🎯 第 5–8 周重点打磨飞行手感——物理参数不是为了"数值正确"，是为了闭眼盲测也能感觉对
- 爆炸特效是答辩门面——粒子数量、扩散速度、颜色渐变都要调
- Canvas Scaler 用 Scale With Screen Size，参考分辨率 1920×1080
- 所有 UI 文字用 TextMeshPro，比原生 Text 清晰三倍

**成果清单**

- 完整 2D 空战场景（视差背景 + Tilemap + 粒子特效）
- 战机物理操控手感达标（盲测认可率 ≥ 80%）
- 全套 UI 界面（含雷达小地图）
- 素材库统一规范文档

**副方向**：前端 A 兼 AI 联调（配合行为树参数可视化），前端 B 兼网络联调（客户端消息收发、延迟显示）

### 方向二：AI · 算法（1–2 人）

**日常职责**

- 威胁感知：威胁网格计算模块，每帧输入敌机位置、弹幕密度、导弹追踪状态，输出综合威胁值
- 行为决策：行为树框架（巡逻→搜索→追击→攻击→规避）五个节点的条件判断与优先级
- 路径规划：
  - BFS：区域巡逻航线生成、目标搜索
  - A*：威胁感知追击路径、低风险规避路径
  - 遗传算法（GeneticSharp 库）：染色体编码（目标编号+进入距离+脱离时机）、适应度函数（伤害比×存活时间×命中率）、交叉率/变异率调优
- 难度配置：三档参数表——入门档 BFS 搜索半径大 AI 反应慢，困难档遗传算法种群 50 个体迭代 30 代
- 测试工具：批量对战命令行脚本，自动记录每局数据输出 CSV，Python 脚本画胜率矩阵热力图

**遗传算法落地路线**

1. 跑通 GeneticSharp 官方 TSP 旅行商 demo，理解染色体、适应度、选择、交叉、变异的 API
2. 把空战攻击目标选择建模：基因为"攻击目标编号、进入角度、脱离距离"三个整数的数组
3. 适应度函数 = (造成伤害 / 自身受伤) × 存活秒数 × (命中次数 / 开火次数)
4. 对比实验：同一初始战场，遗传算法 vs 随机选择 vs 最近目标，各跑 50 局统计胜率

**每周额外关注**

- 🎯 行为树先画 Visio 流程图再写代码——五个状态的切换条件必须全员理解一致
- 威胁网格不能拖帧——每帧计算量超过 1ms 就优化：网格分辨率从 100×100 降到 50×50，用空间哈希
- 数学系组员的核心价值：验证适应度函数的统计显著性，算样本量下限（270 局够不够得出结论）
- 每档 AI 各配一个"性格参数文件"——攻击性、规避倾向、命中精度，方便调参不必重编译

**成果清单**

- 威胁网格计算模块（≤ 1ms/帧）
- 行为树框架（五个状态完整切换）
- 三档 AI 可对战（入门/中档/困难行为可感知差异）
- 胜率矩阵热力图（入门:中档:困难 胜率比）
- 遗传算法对比实验报告

**副方向**：兼前端行为可视化——AI 当前状态显示在屏幕上（巡逻/追击/攻击），便于调试和答辩演示

### 方向三：网络 · 后端（1-2 人）

**日常职责**

- 网关服务器：TCP 长连接管理、客户端登录认证、消息路由转发（收到谁的消息转给谁）
- 逻辑服务器：碰撞权威判定（服务器算碰撞，客户端只展示结果）、战斗规则计算、状态同步广播
- 消息协议：三类 JSON 消息完整定义——操作指令（移动、开火、锁定）、状态同步（位置、血量、分数）、事件通知（击杀、死亡、掉线）
- 房间系统：创建（房主设定模式+地图）、加入（房间列表+密码）、退出、观战（只收状态不发操作）
- 断线重连：心跳包每 3 秒发一次，10 秒无心跳踢出房间，重连后服务器下发全量状态恢复
- 复盘功能：每局自动记录全帧状态到 `.replay` 文件（二进制格式，3600 帧 ≤ 5MB），支持回放与五个关键事件标注

**五个复盘关键事件（已定，后端实现）**

| 事件 | 触发条件 | 时间戳来源 |
|------|---------|-----------|
| 击毁敌机 | 服务器判定敌机血量归零 | 逻辑服务器 |
| 被击毁 | 服务器判定玩家血量归零 | 逻辑服务器 |
| 血量危险 | 血量 ≤ 25% | 逻辑服务器 |
| 导弹锁定 | 导弹追踪状态激活 | 逻辑服务器 |

**Mirror 网络库集成路线**

3. 正式项目用 Mirror——状态同步、断线重连 Mirror 已封装，不用自己写

**每周额外关注**

- 🎯 第 6 周局域网对战是你的高光周——那周你是主角，所有人都配合你联调
- 消息协议是所有方向的接口文档——前端和 AI 组必须能看懂你的协议字段
- 用 clumsy 工具人为注入 100ms/200ms/500ms 延迟和 5% 丢包，观察客户端表现
- 二进制 `.replay` 文件格式：帧号(4B) + 玩家数量(1B) + 每个玩家的位置(8B)+速度(8B)+血量(1B)+武器状态(1B)，一帧约 50 字节，3600 帧 ≈ 180KB——远低于 5MB 预算

### 方向四：美术 · 品控（1 人，可由前端某一人兼任）

**日常职责**

- Asset Store 采购（预算 400–600 元，优先级：粒子特效包 > 天空盒 > 战机精灵 > 音效）
- 开源素材收集（OpenGameArt、itch.io 免费 2D 战机、Kenney.nl 全套免费素材）
- 全组素材风格审核：色调统一（冷色调空战主题）、像素密度一致（PPU 100）、比例协调
- Sprite 导入后处理：切片、Pivot 点（战机中心、子弹前端）、Sprite Atlas 打包（减少 Draw Call）
- 粒子特效：爆炸（橙色→灰色渐变，持续 0.5 秒，30 粒子）、导弹尾焰（白色→蓝色渐变，持续追踪）、引擎喷射（持续粒子，蓝色，大小随油门变化）

**每周额外关注**

- 🎯 第 3 周开始收素材——五个人的打砖块不能五种颜色的砖
- 答辩视觉印象占比极高——爆炸特效和视差背景是门面，预算优先花在这里
- 粒子特效不要贪多——三个特效（爆炸、尾焰、引擎）调好就够了
- 颜色统一：玩家蓝色系、敌方红色系、中立灰色系

**成果清单**

- 素材库（素材来源+授权说明文档）
- 三套粒子特效（爆炸、导弹尾焰、引擎喷射）参数定稿
- 全组游戏视觉风格统一审核通过

**副方向**：兼前端场景搭建——素材最终要落进 Unity Scene，不会调 Sorting Layer 和渲染层级等于白做
