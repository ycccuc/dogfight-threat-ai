# 团队学习路线与周目标


## 概览

```
 6月           7月           8-9月         10-11月        12-2月           3-5月
  │             │             │              │              │               │
  ├─ 基础训练期 ─┤             ├──正式开发期──────────────────────────────────┤
  │             │             │              │              │               │
  6个小游戏    2个整合项目    单机核心闭环    AI+前后端      联机上线       收尾交付
```

---

## 第一阶段：基础能力建设（6 月 · 4 周）

> 每周两条线并行：**游戏开发训练** + **需求调研任务**
> 月末合并为完整需求分析报告。

---

### 第 1 周（6/7–6/13）：环境就绪 + 第一个程序

**游戏：移动方块**

一个彩色方块精灵，按 WASD 在场景中移动，碰到边界停止。

| 学会的技能 | 对应立项书模块 |
|-----------|---------------|
| 创建工程、认识 Scene / Game / Hierarchy / Inspector | 引擎基础 |
| 导入图片为 Sprite，拖入场景 | 素材管理 |
| 第一个 C# 脚本：`Input.GetAxis` + `Transform.Translate` | 战机操控 |
| `Update()` 生命周期 | 每帧逻辑 |
| 提交到 GitHub | 版本控制 |

**需求工作**

- 每人了解几款 2D 空战游戏，用表格记录：操控方式、AI 表现、联机支持
- 周末汇总为《竞品体验记录表》

**周成果**

- 每人一个可运行的"移动方块"工程，推送到自己的 feature 分支
- 5 份游戏体验记录 + 汇总表
- 国创赛平台报名完成

---

### 第 2 周（6/14–6/20）：物理与碰撞

**游戏：弹球对战**

屏幕左右各一个挡板，球在中间反弹。挡板由两个玩家分别控制上下移动。球穿过某侧边界，对手得分。先到 5 分获胜。

| 学会的技能 | 对应立项书模块 |
|-----------|---------------|
| Rigidbody2D + 质量、阻力、重力 | 飞行动力学 |
| BoxCollider2D + 碰撞检测 | 碰撞系统 |
| `OnCollisionEnter2D` 事件 | 伤害计算 |
| UI Text 显示比分 | 抬头显示 |
| Physics Material 2D（弹力） | 物理参数调校 |

**需求工作**

- 整理竞品对比矩阵：从操控手感、AI 智能程度、联机体验三个维度打分
- 确定本项目差异化方向（哪些是竞品做不好的，我们来做）

**周成果**

- 弹球对战可运行，有计分和胜负判定
- 《竞品分析报告》（`docs/requirements/竞品分析.md`）
- 功能边界初稿：做什么、不做什么

---

### 第 3 周（6/21–6/27）：批量管理与 UI

**游戏：打砖块**

顶部排列 5 列 × 3 行砖块（Prefab 预制体）。底部挡板由玩家控制。球碰到砖块→砖块销毁+加分。全部消除→胜利。球落地→失败。显示分数和剩余球数。

| 学会的技能 | 对应立项书模块 |
|-----------|---------------|
| Prefab 预制体的创建与复用 | 子弹/导弹/敌机管理 |
| `Instantiate` + `Destroy` | 武器发射与销毁 |
| 循环生成 + 网格布局 | 敌机波次 |
| Canvas + Text + Button | 主菜单界面 |
| 场景切换（`SceneManager`） | 游戏流程 |

**需求工作**

- 梳理六大核心功能模块的详细描述
- 画系统用例图（三类角色：玩家、AI、观战者）
- 定义第一版前后端消息协议：操作指令有哪些、状态同步哪些字段

**周成果**

- 完整打砖块：开始→游玩→胜利/失败→重开
- 《功能需求清单》（`docs/requirements/功能需求.md`）
- 系统用例图 + 消息协议草案

---

### 第 4 周（6/28–7/4）：完整射击游戏 + 需求报告交付

**游戏：飞机射击**

玩家战机在底部用左右键移动，空格发射子弹。敌机从顶部定时生成并向下移动。子弹击中敌机→双方销毁+加分。敌机撞到玩家→扣血。血量为零→游戏结束。

| 学会的技能 | 对应立项书模块 |
|-----------|---------------|
| 子弹 Instantiate + 自动移动 + 出界销毁 | 武器系统完整流程 |
| 碰撞层（Layer）设置子弹/敌机/玩家 | 分层碰撞检测 |
| 协程（Coroutine）定时生成敌机 | 波次管理 |
| 血条 UI + 分数 UI | 抬头显示系统 |
| PlayerPrefs 存储最高分 | 数据持久化 |

**需求工作**

- 将前三周产出整合为《需求分析报告》
- 完成空战威胁评估模型独立脚本（控制台程序，输入坐标输出威胁值）
- 完成 A* 寻路独立脚本（网格地图上从 A 到 B 绕过障碍）

**周成果**

- 完整飞机射击游戏，有血量、分数、结束画面
- **《需求分析报告》定稿** → 提交给老师审阅
- 威胁评估 + A* 算法原型脚本 → 验证可行性
- 全组成员完成 6 次 Git 提交 + PR 合并的完整流程

## 第二阶段：原型验证与项目启动（7 月 · 4 周）

> Unity 基础已就位。7 月起技能训练向项目实际需求靠拢，同时搭建正式工程骨架。

---

### 第 5 周（7/5–7/11）：智能敌人

**游戏：AI 追击者**

在第 4 周飞机射击基础上，新增一种敌机类型。该敌机不沿固定航线，而是每帧计算玩家位置并向玩家移动。当玩家血量低于一半时加速追击，高于一半时与玩家保持距离。

| 学会的技能 | 对应立项书模块 |
|-----------|---------------|
| 向量计算（`Vector2.Distance`、方向） | 敌我距离与方位 |
| 简单状态机：巡逻 ↔ 追击 | 行为树基础 |
| 条件判断驱动的行为切换 | 动态决策 |

**项目工作**

- 创建正式的 Unity 2D 工程 `src/client/DogfightThreatAI/`
- 导入战机精灵、背景素材，搭建基础空战场景
- 开始飞行动力学原型：油门/惯性/阻尼参数初调

**周成果**

- AI 追击者游戏可运行（敌机行为有差异）
- 正式 Unity 工程建立，战机能在场景中物理飞行

---

### 第 6 周（7/12–7/18）：网络通信

**游戏：局域网对战**

两人各自打开程序，一台作为主机，另一台连接。各自控制一架方块战机移动并射击。对方的位置和子弹能实时显示在自己屏幕上。

| 学会的技能 | 对应立项书模块 |
|-----------|---------------|
| TCP Socket 连接建立 | 客户端连接管理 |
| JSON 序列化/反序列化 | 消息协议 |
| 消息收发循环 | 网络通信 |
| 主机/客户端模式 | C/S 架构 |

**项目工作**

- 完成飞行动力学全部参数调校（惯性系数、转向阻尼、速度上下界）
- 搭建网关服务器工程框架（C# 控制台 + Socket 监听）
- 设计正式消息协议（操作指令、状态同步、事件通知三类）

**周成果**

- 局域网双人对战可运行，双方能看到对方移动
- 飞行动力学手感到位
- 网关服务器可接受连接并转发消息

---

### 第 7 周（7/19–7/25）：整合——单机空战原型

**目标：前 6 周技能的融合应用**

将移动、物理、武器、AI、网络五个技能模块整合为一个完整的 2D 空战原型：

- 多层视差背景（至少两层不同滚动速度）
- 战机移动控制（Rigidbody2D 物理驱动，惯性+阻尼）
- 机炮射击（子弹生成+射线检测+碰撞销毁）
- 一档基础 AI 敌机（行为树：巡逻→搜索→追击→攻击→规避）
- 得分 + 血量 + 游戏结束

**项目工作**

- 实现行为树框架（五个状态的节点结构和切换条件）
- 以 BFS 实现 AI 巡逻与目标搜索
- 对象池管理（子弹复用，避免频繁 Instantiate/Destroy）

**周成果**

- 单人 vs AI 空战原型：可操控战机、可射击、AI 会追你、有胜负
- 行为树框架代码
- 对象池运行正常

---

### 第 8 周（7/26–8/1）：第一阶段验收

**任务**

- 全栈走通一次完整流程：客户端飞行+射击 → 碰撞检测 → 得分/HUD 更新 → AI 响应
- 修复第一阶段的已知问题
- 补齐缺失的文档章节

**周成果**

- 可运行的单机空战原型
- 第一阶段总结报告
- 技术文档更新至最新
- GitHub 分支合并，`dev` 分支可运行

## 第三阶段：正式项目开发（8 月–次年 5 月，按立项书执行）
